## 🏗️ Repository Structure

```text
.
├── 0. BOOTSTRAP      # Bootstrap을 활용한 UI 레이아웃 및 반응형 웹 실습
├── 1. HTML           # 웹 표준 및 시맨틱 태그 등 마크업 기초
├── 2. NODE           # Node.js 환경 구성 및 서버 사이드 기초
├── 3. VUE            # Vue.js (Pinia, Composition API) 기반 프론트엔드 실습
└── 04_JAVA           # JDK 17 기반 객체지향 프로그래밍 및 자바 심화
```

## 📝 Commit Convention (Udacity Style)

프로젝트의 일관성과 협업을 위해 아래의 Udacity 커밋 메시지 규칙을 적용합니다.

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정 (README 등)
- `style`: 코드 의미에 영향을 주지 않는 변경사항 (공백, 포맷팅, 세미콜론 등)
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드 추가 및 수정
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등
